# Быстрый старт

## Установка зависимостей

```bash
npm install
```

## Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

## Сборка для production

```bash
npm run build
```

Собранные файлы будут в папке `dist/`

## Запуск через Docker

```bash
# Сборка образа
docker build -t dobro-v-max-frontend .

# Запуск контейнера
docker run -p 80:80 dobro-v-max-frontend
```

## Интеграция с MAX

1. Для работы с MAX UI раскомментируйте импорты в `src/main.jsx`:
   ```javascript
   import { MaxUI } from '@maxhub/max-ui'
   import '@maxhub/max-ui/dist/styles.css'
   ```

2. MAX Bridge SDK автоматически загружается из `index.html`

3. Настройте переменные окружения в `.env`:
   ```
   VITE_API_URL=http://your-backend-url/api
   ```

## Структура приложения

- **Главная страница** (`/`) - обзор платформы
- **Каталог** (`/catalog`) - просмотр всех заявок с фильтрацией
- **Создать заявку** (`/create-request`) - форма создания новой заявки
- **Регистрация организации** (`/register-org`) - регистрация НКО
- **Детали заявки** (`/request/:id`) - подробная информация и отклик
- **Чат** (`/chat/:id`) - общение с организацией/волонтёром
- **Профиль** (`/profile`) - личный кабинет пользователя

## Примечания

- Приложение использует моковые данные для демонстрации
- Для работы с реальным бэкендом настройте `src/utils/api.js`
- Все API вызовы должны быть реализованы на бэкенде
